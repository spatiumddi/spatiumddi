# NTP Feature Specification

## Overview

SpatiumDDI can manage NTP (Network Time Protocol) servers as part of the overall network services stack. NTP management is optional but integrated: NTP server addresses can be pushed to DHCP clients via DHCP option 42, configured per-subnet, and monitored alongside DHCP and DNS from the same admin dashboard.

SpatiumDDI does not implement an NTP server itself — it manages existing NTP daemons (**Chrony** preferred, **ntpd** legacy supported).

---

## 1. NTP Server Model

```
NTPServer
  id, name, description
  driver: enum(chrony, ntpd)
  host, ssh_port (for management)
  ssh_credentials (encrypted)
  roles: [enum(primary_stratum, secondary_stratum, pool_member)]
  stratum: int (1–15, read from server)
  upstream_servers: str[]       -- upstream NTP sources (e.g., pool.ntp.org, PPS source)
  serve_clients: cidr[]         -- which clients this server accepts queries from
  is_enabled: bool
  status: enum(online, offline, degraded)
  last_health_check_at: timestamp
  current_offset_ms: float      -- read from server, displayed in UI
  current_stratum: int          -- read from server
  reference_id: str             -- upstream reference (e.g., GPS, PPS, upstream IP)
```

### NTP Server Groups

Same group concept as DHCP/DNS — group multiple NTP servers for redundancy:

```
NTPServerGroup
  id, name, description
  servers: [NTPServer]
  is_default: bool              -- used when subnets don't specify a group
```

## 2. Integration with DHCP (DHCP Option 42)

NTP server IPs are pushed to DHCP clients via:
- **DHCP Option 42** (RFC 2132) — NTP server IP list
- **DHCP Option 100** — POSIX timezone string (optional)
- **DHCP Option 101** — TZ database timezone name (optional)

Configuration per subnet:

```
Subnet.ntp_servers: inet[]        -- explicit NTP server IPs (overrides group)
Subnet.ntp_server_group_id        -- use IPs from this NTP group
Subnet.ntp_timezone: str          -- IANA timezone, e.g., "America/New_York"
```

If neither is set, the global platform NTP server list is used.

```python
    async def set_allowed_clients(self, cidrs: list[str]) -> None:
        # Update allow directives in chrony.conf, reload
```

**Chrony config template** (generated by driver):
```
# Generated by SpatiumDDI — do not edit manually
server 0.pool.ntp.org iburst
server 1.pool.ntp.org iburst
server 2.pool.ntp.org iburst

allow 10.0.0.0/8
allow 192.168.0.0/16

makestep 1.0 3
rtcsync
logdir /var/log/chrony
```

Changes apply via `chronyc reload sources` — **no daemon restart required**.

### ntpd Driver (Legacy)

Manages `/etc/ntp.conf` via SSH. Changes require `systemctl reload ntp` (SIGHUP — not a full restart).

## 4. NTP Monitoring

The Health Dashboard NTP panel shows per-server:
- Stratum
- Reference source (upstream IP or GPS/PPS indicator)
- Current offset (ms) — color coded: green < 10ms, amber < 100ms, red > 100ms
- Number of clients synchronized (from `chronyc clients`)
- Upstream server reachability

Prometheus metrics:
```
spatiumddi_ntp_stratum{server_id}
spatiumddi_ntp_offset_ms{server_id}
spatiumddi_ntp_reference{server_id}       -- label with reference ID
spatiumddi_ntp_clients_total{server_id}
spatiumddi_ntp_server_status{server_id}   -- 1=online, 0=offline
```

## 5. NTP Permissions

NTP follows the same permission pattern:

| Role | Capability |
|---|---|
| **superadmin** | Full NTP server and group management |
| **admin** | View NTP status; cannot modify server config |
| **operator** | View NTP status only |
| **viewer** | View NTP status only |

NTP server configuration (upstream sources, client ACLs) is superadmin-only — it's platform infrastructure, not delegatable to subnet-level admins.

---

## 6. NTP and the Appliance

When running as an OS appliance:
- `chrony` is installed and enabled by default
- The appliance can optionally **serve time** to internal clients (enabled in first-boot wizard)
- The appliance NTP server is automatically added to the default NTP server group
- DHCP scopes created on the appliance will have the appliance's IP as `ntp-servers` option by default
